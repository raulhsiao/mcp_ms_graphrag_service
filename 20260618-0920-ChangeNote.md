- 日期：2026-06-18 09:20

## 本次重點：多資料庫支援改為「參數驅動」

讓每次查詢以工具參數指定要使用哪個內建資料庫，取代先前的全域切換狀態。

### 內建資料庫

| 名稱 | 路徑 |
| ---- | ---- |
| `Airoha`（預設） | `/workspace/DB_AIROHA` |
| `MS GraphRAG` | `/workspace/DB_MS_GRAPHRAG` |

### 移除

- `graphrag_select_database` tool。
- `_active_root` 全域變數與 `GRAPHRAG_DB_ROOT` 環境變數的初始邏輯。

### 新增 / 修改

- 三個查詢工具各自帶 `database` 參數（預設 `"Airoha"`），彼此獨立、無共享狀態：
  - `graphrag_query(query, method, community_level, database)`
  - `graphrag_index_status(database)`
  - `graphrag_list_entities(top_n, database)`
- `_resolve_root(database)`：依名稱查 `BUILTIN_DATABASES`；未知名稱 raise `ValueError`、目錄不存在 raise `FileNotFoundError`。
- `_run_graphrag_cli(args, cwd, ...)`：改為接收呼叫端解析好的 `cwd`，不再自行解析。
- 每個工具回傳 JSON 都加上 `database` 欄位，方便追蹤查的是哪個庫。

### 順帶修掉的瑕疵

舊版 `_resolve_root()` 在 CLI 的 try 區塊內被呼叫，目錄不存在時會被誤報成「找不到 graphrag 指令」。驗證移到呼叫端後，錯誤訊息正確（未知名稱在執行 CLI 前就擋下並回 `success=false`）。

### 好處

天生 stateless —— 無跨請求/跨 process 共享狀態，相容 `stateless_http=True` 與未來多 worker 水平擴展；同一 agent 可在連續查詢中切換不同資料庫而互不干擾。

## 本次 session 其他變更（依序）

1. **修正 `mcp.run()` 啟動錯誤**
   `FastMCP.run()` 不接受 `host`/`port`，改在 `FastMCP(...)` 建構時設定，`run()` 只留 `transport="streamable-http"`。

2. **讓 agent 連線即自動了解服務用法**
   加入 `instructions`（MCP `initialize` 握手時回傳，client 會注入 system prompt）：涵蓋服務概觀、建議流程、`method` 選擇與注意事項。

3. **新增 Resource 與 Prompt**
   - Resource `guide://graphrag`（`text/markdown`）：完整使用手冊（內建資料庫、工作流程、`method` 對照、範例、錯誤排解），供 agent 按需讀取。
   - Prompt `summarize_knowledge_base(database)`：引導用 global 查詢做結構化總結。
   - Prompt `ask_about_topic(topic, database)`：針對指定主題提問並建議 `method`。

   三者皆已與 `database` 參數同步。

## 生效方式

修改 server 後需 **重啟 server 並重新連線**（instructions / resources / prompts 僅在 `initialize` 時取得）：

```shell
python ./mcp_ms_graphrag/mcp_ms_graphrag_server.py
```
